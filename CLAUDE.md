# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A Claude Code plugin that speaks short TTS summaries when Claude finishes a task or needs user attention. Uses Microsoft edge-tts neural voices by default (via `uvx`, online) with macOS `say` as the offline fallback. Distributed via the `usterk/simple` marketplace.

## Architecture

- **hooks/hooks.json** — auto-registers all hooks via `${CLAUDE_PLUGIN_ROOT}` when the plugin is enabled; no settings.json edits, no file copying
- **hooks/tts_utils.py** — shared module: config loading (`~/.claude/simple-tts-config.json`), phonetic sanitization, non-blocking `speak()`, transcript TTS tag extraction, voice→gender + gender→edge-voice mapping (`voice_gender`, `EDGE_VOICES`, `edge_voice_for`). Speech state (PID + timestamp of the running TTS process) lives in `~/.claude/simple-tts-state.json` under `flock`. **Global speech queue**: while TTS is busy, a non-priority message no longer gets dropped — it's written to a filesystem queue, `~/.claude/simple-tts-queue.d/` (one `<time_ns>-<pid>.json` file per entry, holding the fully-built payload), `QUEUE_TTL_SECS=40` (stale entries are skipped on pop), `QUEUE_MAX=8` (oldest entries evicted past that). Primitives `_queue_enqueue`/`_queue_pop`/`_queue_clear` have no lock of their own — every call happens inside the existing state-file flock (`_locked_state`), so the queue and the PID/ts state stay consistent under one lock. `parse_tts_tag`/`TTS_TAG_RE` now parse an optional category out of the tag (`<!-- TTS[ok|err|q]: -->`, lowercase only, no `re.IGNORECASE` — an unrecognized bracket like `[foo]` fails to match at all rather than falling back), returning `(category, text)`; `extract_tts_from_transcript` returns that same `(category, text)` tuple or `None`. `_project_label`/`_should_announce_project`/`fresh_busy_count` back the optional project-name prefix (see `speak()` below). New `DEFAULT_CONFIG` keys: `announce_project` (`auto`|`on`|`off`, default `auto`) and `earcons` (bool, default `true`)
- **hooks/edge_speak.py** — detached helper, now used for **both** engines: `engine="edge"` synthesizes via `uvx edge-tts` (no install — runs out-of-process, plugin stays stdlib-only) and falls back to local `say` on any failure (offline, no `uvx`, timeout, empty result); `engine="say"` renders straight through `say` (`_say`/`_say_raw`/`_say_to_file`). Routing `say` through this helper too (instead of `speak()` spawning `say` directly) means both engines share the same queue-drain loop below — `speak_cli`/MCP `server.py` inherit this automatically since they call `speak()`. Plays the mp3 with `afplay`. Reads its payload from the `SIMPLE_TTS_PAYLOAD` env var (so text never appears in `ps`). Launched as its own process group so a priority interrupt kills synthesis + playback together via `os.killpg`. On a cache hit it plays straight from disk (skips synthesis); on a miss it synthesizes to a `.synthtmp-*.mp3` temp in the cache dir, `os.replace`s it into place (atomic), records usage and evicts. All cache logic lives in `audio_cache.py`. The `say` fallback is never cached. **Speech envelope for the overlay** (`_compute_envelope`→`_write_speak_state`, state file `~/.claude/simple-tts-speak.json`): when the overlay is enabled (`payload["envelope"]`, set from `knight_rider` in `speak()`), `_play_speech` writes the RMS envelope of the *plain voice* so the overlay animates the "speaking" eye in the rhythm of what's audible. Computed for **every** theme, not just KITT — the old gate was `intro_sound != none` (siren-on), which meant spark/cylon (no siren) never got an envelope and the eye fell back to the host's synthetic `sin` pulse. Edge caches the envelope via `_cached_envelope` (`audio_cache.read_env`/`store_env` sidecar → no ffmpeg pass on cache hits); `say` computes it live. When a KITT mix plays, the envelope is padded with `_INTRO` s of silence so the modulator starts with the voice, not the siren intro. **Background sound** (`intro_sound` config, default `"kitt"`): when set, `_mix_kitt` ffmpeg-mixes the speech under `sounds/<name>.mp3` at *playback* — 1 s intro, sidechain-ducked bed, outro ending in the sound's quiet valley (KITT loops in ~1.28 s cycles; end = first valley at `0.04 + n·1.28` after intro+speech+0.35 s, with a 0.10 s fade to zero). Mix goes to a system temp mp3, played then unlinked; the on-disk **cache always holds the plain speech**, so the effect toggles instantly with config and never pollutes the cache. Missing ffmpeg/ffprobe or sound file, or any ffmpeg error → plain speech plays. A missing `intro_sound` key means off (production always passes it explicitly). **Earcon**: `_play_earcon` plays `hooks/sounds/earcon_<ok|err|q>.mp3` first, blocking, before the speech (for both engines) when the payload carries an `"earcon"` key; missing file or unrecognized category is silently skipped. **Queue drain**: after playing the payload it received via env, `_drain_loop` keeps calling `_speak_payload` for whatever `_drain_step` pops off the global queue (`tts_utils._queue_pop`), 0.4 s of breathing room between messages, all under the same state flock as `speak()`. It drains only as long as the state still belongs to this process (`state["pid"] == os.getpid()`) — if something else claimed the state (a priority interrupt, another spawn), it stops immediately without touching state. Each queue entry is unlinked under the lock before being returned, so it's replayed exactly once even if this helper is later hard-killed; an empty queue leaves state as `{"ts": ...}` with no `pid`, signalling nobody is speaking. `QUEUE_TTL_SECS`/priority's `_queue_clear()` are what clean up after a helper that got killed mid-drain
- **hooks/audio_cache.py** — the on-disk audio cache (used by edge_speak + cache_cli). Entries are `<sha256(engine,voice,rate,text)>.mp3` under `~/.claude/simple-tts-audio-cache/`; a flock-guarded `index.json` holds per-entry metadata (text, voice, rate, play count, created/last_used). **Eviction is size-based** (`cache_max_mb`, default 100): when total mp3 size exceeds the budget, entries are removed least-used-then-oldest (`(plays, last_used)` ascending), only as many as needed. Reconciles index↔disk (drops rows whose mp3 vanished; mp3s with no row count as plays=0 → evictable). `sweep_temp()` removes `.synthtmp-*` orphans older than `TMP_MAX_AGE`. **Envelope sidecars**: the overlay's speech modulator (see edge_speak) is content-addressed the same way — `<hash>.env.json` (`{dt, env}`) sits next to `<hash>.mp3`, computed once and reused on cache hits (`read_env`/`store_env`). `.env.json` files are never counted as cache entries (`_disk_entries` only sees `.mp3`); eviction and `clear` remove a row's sidecar alongside its mp3, and `evict` also prunes orphan sidecars whose mp3 vanished (`_prune_orphan_envs`). API: `key_for`, `cache_path`, `env_path_for_key`, `read_env`, `store_env`, `record_store`, `record_hit`, `evict`, `stats`, `clear`, `sweep_temp`
- **hooks/cache_cli.py** — `python3 hooks/cache_cli.py stats|prune [--max-mb N]|clear`; backs `/tts cache`. `stats` prints each phrase with play count, last-used and size vs budget; `prune` evicts to budget; `clear` wipes the cache
- **hooks/stop_tts.py** — Stop hook (tag mode only): extracts `<!-- TTS: message -->` (or the categorized `<!-- TTS[ok|err|q]: message -->`) from `last_assistant_message` (preferred, via `extract_tts_from_message`) or transcript fallback. Both now return `(category, text)`. No tag → silence (or `fallback_message` from config, with `category=None`). Exits immediately when `speak_via == "tool"`. Passes `project=os.path.basename(cwd)` and the extracted `category` through to `speak()` — this is the only hook that ever sets `category` (tag mode only; `notification_tts`/`ask_question_tts` speak with `priority=True` instead and never carry an earcon)
- **hooks/notification_tts.py** — Notification hook: speaks a short phrase in the configured language (`MESSAGES` catalog), naming the tool from permission messages when possible. Never reads transcript (avoids repeating stale messages). **Busy filter**: `permission` notifications are always spoken (a real decision); a non-permission (idle) `waiting` notification is suppressed — no speech, no `attention` marker — while `session_busy_fresh()` says the session's turn is still in progress (the agent is working, e.g. waiting on subagents, and the notification is a false "waiting"). Passes `project=os.path.basename(cwd)` to `speak()` (no `category` — priority speech, no earcon)
- **hooks/ask_question_tts.py** — PreToolUse hook, matcher `AskUserQuestion|ExitPlanMode`: speaks the actual decision content immediately when the agent asks it, instead of waiting for the idle "waiting" notification. For `AskUserQuestion` it reads the first question text out of `tool_input.questions` (truncated to `MAX_LEN`, "I have a few questions, first: …" when there's more than one); for `ExitPlanMode` it speaks a fixed "plan ready, approve?" phrase. Speaks with `priority=True` and sets the session's `attention` marker — this is the real user-actionable event the busy filter above is relying on. Also passes `project=os.path.basename(cwd)` (no `category`). Silent no-op without config, for other tools, or when there's no question text to read
- **hooks/session_start.py** — SessionStart hook: injects the TTS instruction as `additionalContext`, generated from config (language, voice gender → grammar forms, name). Two variants by `speak_via`: `tag` (append a hidden `<!-- TTS: -->` comment) or `tool` (call the `speak` MCP tool, no visible marker). In `tag` mode the instruction also teaches the model the categorized forms — `<!-- TTS[ok]: ... -->` for success/done, `<!-- TTS[err]: ... -->` for an error/blocker, `<!-- TTS[q]: ... -->` when a decision is needed, no brackets for neutral, always lowercase — and notes that the category plays a short matching earcon before the speech (`build_tag_instruction`). Shared content rules via `content_rules()`. Replaces the old CLAUDE.md-append approach
- **hooks/message_display.py** — MessageDisplay hook (tag mode): rewrites the hidden `<!-- TTS: ... -->` (or `<!-- TTS[ok|err|q]: ... -->`) marker in the on-screen display via `hookSpecificOutput.displayContent`. Display-only — the transcript keeps the marker so the Stop hook still speaks it. Default renders the spoken line as a `🔊 <text>` in a category color (ANSI): `GREEN` for `ok` or no category (today's neutral tag), `FIREBRICK` for `err`, `AMBER` for `q`; config key `tag_display` switches it: `styled` (colored speaker, default) / `plain` (🔊 line, no ANSI — fallback if a terminal mangles ANSI, no color distinction) / `hidden` (remove the marker entirely). Field-name-agnostic (the MessageDisplay stdin schema is undocumented): it locates the marker anywhere in the JSON; emits nothing when no marker is present. The regex is unanchored on the category bracket (`TTS(?:\[(ok|err|q)\])?`) so both tagged and legacy untagged markers match, and the prefilter (`_find_text_with_marker`) looks for `"<!--"` + `"TTS"` case-insensitively so it still catches categorized tags. MessageDisplay fires per rendered chunk and the marker sits at the message end, so it is rewritten when the final chunk renders. Solves the visible-`<!-- TTS -->` problem (some Claude Code versions render HTML comments as raw text) without losing tag mode's deterministic speech
- **hooks/phonetics/<lang>.json** — phonetic dictionaries (English term → phonetic spelling); user overrides merge in from `~/.claude/simple-tts-phonetics.json`
- **hooks/speak_cli.py** — manual test mode: `python3 hooks/speak_cli.py "text"` speaks through the full pipeline, bypassing mute and quiet hours (`force=True`)
- **skills/tts/SKILL.md** — invoked as `/simple-tts:tts` (plugin skills are namespaced; a bare `/tts` needs a personal `~/.claude/commands/tts.md` wrapper). Subcommands: `on|off|status`, `theme [name|off]` (picks `overlay_theme` and toggles `knight_rider` — a real theme enables the overlay, `off` hides it; no arg lists spark/kitt/cylon + current), `howl [on|off|auto]`, `distortion [on|off|auto]`, `cache [stats|prune|clear]`. All config reads/writes go through **`hooks/config_cli.py`** (`get`/`set`/`show`) instead of inline python; `cache` goes through `cache_cli.py`. Both resolve their script path whether installed as a plugin or run from the repo. There is no separate `knight-rider` subcommand — the overlay is controlled via `theme`
- **hooks/config_cli.py** — `python3 hooks/config_cli.py get <key> [default] | set <key> <value>... | show`; the single config read/write helper backing `/simple-tts:tts` (stdlib-only, reuses `tts_utils.CONFIG_PATH`/`DEFAULT_CONFIG`/`load_config`). `set` writes one or more `key value` pairs atomically with type inference (`true`/`false`→bool, ints→int, else string) and refuses to run when the config file is absent (exit 2 → run `/simple-tts-setup`); `show` prints the effective merged config for `/tts status`
- **mcp/server.py** + **.mcp.json** — stdlib-only MCP stdio server with a `speak` tool, for environments without hooks (Cowork, desktop chat) and for Claude Code `tool` mode. Reuses `tts_utils.speak()` (so mute + quiet hours apply); when to call it is governed by the SessionStart instruction, not the tool description
- **skills/simple-tts-setup/SKILL.md** — setup wizard (`/simple-tts-setup`): writes the config file, handles uninstall and migration from pre-2.0 installs
- **hooks/user_prompt.py** — UserPromptSubmit hook: marks "this session is working" for the overlay (per-session marker in `~/.claude/simple-tts-busy.d/`, `set_session_busy`) and clears the session's `attention` marker (user is back). The Stop and SessionEnd hooks clear both. Silent no-op without config; writes nothing to stdout so it adds no prompt context
- **hooks/session_end.py** — SessionEnd hook: on a clean session close (window closed / `/exit`) clears both this session's `busy` and `attention` markers, so the overlay doesn't keep showing work / blue "waiting" for a gone session. Hard kills fall back to the stale windows (15 min busy / 10 min attention). Silent no-op without config
- **hooks/attention_clear.py** — PostToolUse hook, two jobs on every tool call (of the main agent AND of its subagents, which run their own PostToolUse hooks): (1) **heartbeat** — refresh this session's `busy` marker (`set_session_busy(session_id, True)`) so the overlay's `think` mode stays alive as long as tools keep running. Without it the marker is only stamped once at UserPromptSubmit and cleared at Stop, so a session that hands work to subagents (or a background job re-invoked between turns) drops to `idle` while it's still working — the tool boundary is the reliable "still working" signal that survives sub-turn Stop/UserPromptSubmit churn. (2) a tool ran, so any permission was granted — **clear** this session's `attention` marker. (This replaces the old no-attention-markers fast path: the hook now always reads stdin for `session_id`.) The `attention` marker itself is set by the Notification hook (`set_session_attention`, dir `~/.claude/simple-tts-attention.d/`) regardless of mute/quiet hours — the visual is independent of speech
- **hooks/limit_tts.py** — StopFailure hook (registered with matcher `rate_limit|billing_error|max_output_tokens|overloaded`): speaks a short alert when the turn ends on an API limit/error. `StopFailure` fires INSTEAD of `Stop` when the turn is cut short by an API error; the matcher filters on error type. `translate_limit(error_type, msgs)` maps the type to a phrase from the `MESSAGES` catalog (per language), falling back to a generic "limit reached" phrase for an empty/unrecognized type (the `StopFailure` input schema is undocumented, so `error_type` is read defensively). Speaks with `priority=True`; silent no-op without config. Only limit/overload types are matched — auth/network errors don't trigger speech
- **overlay/** — the **status overlay** (optional GUI daemon, not on the hook path so the core stays stdlib-only). `kitt_state.py` (stdlib, unit-tested) aggregates simple-tts state across all sessions into `snapshot()` with **two independent axes**: **`mode`** (movement) — **speak** (TTS PID alive + a real `afplay`/`say` process) > **think** (fresh marker in `simple-tts-busy.d`; stale after 15 min) > **idle**, `None` when `knight_rider` is off — and **`waiting`** (color only) — a fresh marker in `simple-tts-attention.d` meaning someone waits for the user, **stale after 10 min** (no reaction for 10 min ⇒ back to the normal color). Attention no longer drives movement, only the theme's accent color. Plus `busy` (how many sessions work in parallel), `waiting_count` (how many sessions wait for the user — `attention_count()`, so a theme can color exactly that many lenses) and `age` (oldest running work, seconds). Markers are cleared per-session by UserPromptSubmit / PostToolUse / Stop / **SessionEnd** (`session_end.py`, clean exit); the stale windows are the backstop for hard-killed sessions. `kitt_overlay.py` is the theme **host**: transparent, click-through, always-on-top Cocoa `NSPanel` per screen (`CanJoinAllSpaces` + `FullScreenAuxiliary`, exactly menu-bar height), fast/slow timers (after 60 s of continuous idle the panels fade, order out and only a 1 s state poll remains), speech-envelope reading and applying theme updates inside one `CATransaction`. The look itself lives in **overlay/themes/** — `base.py` defines the contract (`sprites()` PIL images → converted once to `CGImage`; `layers()` flat layer specs; `step()` per-frame `(idx, "op"|"pos"|"img", value)` updates with dirty-check, so an unchanged frame costs nothing) plus shared extras every theme gets for free: fleet pips (one dot per working session, up to 5, shown in think/attention) and `age_mult` (animations speed up the longer the oldest work runs). Themes: `spark` (default; central green cat-eye lens with a spark stream — dim dots drift in slowly from the left, accelerate through the lens and shoot out right as color-coded streaks (green/orange/red/blue/violet by agent count); the lens **splits into one lens per working agent** and merges before rotating 90° to speak — the merge-into-speak and the speak rotation are fast so the eye is horizontal before audio starts, never speaking mid-tilt; the rotation tau is **directional** (0.07 into speak, 0.12 back — the return is unchanged), and the eye-scale into speak is quick (0.045) so the transition to speaking never lags the audio; its **attention** turns **one lens per waiting session blue** (driven by snapshot `waiting_count` = fresh markers in `attention.d`, clamped to the visible lens count) — one waiting task ⇒ exactly one blue lens, the other agents' lenses stay green (per-lens `attn_g[]` gate, no longer a single global gate); the spark **stream** turns blue whenever anyone waits (`attn_stream`)), `kitt` (continuous LED sweep, afterglow tail, speech bloom — its **attention** turns KARR yellow, `kitt.KARR`), `cylon` (wider pure-red eye, sinusoidal sweep — attention stays amber, `base.AMBER`). Every theme implements all four modes; **attention** color is per-theme (each theme picks its own accent — no longer a shared amber; blink pattern still from `base.blink`). Selected via the `overlay_theme` config key, switchable live — the host rebuilds layers without restart. `kitt_frame.py` keeps the KITT sprite drawing (used by the kitt/cylon themes). Layer transforms (`xf` op) let a theme rotate+scale a sprite smoothly (spark's eye). Deps (pyobjc Cocoa+Quartz + Pillow) and autostart via `install_overlay.sh` (copies overlay + themes to `~/.claude/simple-tts-overlay/`, writes a LaunchAgent); `uninstall_overlay.sh` reverses it
- **.claude-plugin/plugin.json** — plugin manifest (version auto-bumped by CI)

## Key design decisions

- **Speech engine** (`"engine"` config key, default `"edge"`): `edge` uses Microsoft edge-tts neural voices (online, high quality) with the local `say` voice as automatic fallback; `say` uses the local macOS voice only. The edge voice is **derived at runtime** from `voice_gender(config["voice"])` + language via `EDGE_VOICES` — there is no stored edge-voice field, so it auto-adapts per machine (Krzysztof→Marek, Ewa→Zofia). Defaulting `engine` to `edge` in `DEFAULT_CONFIG` means existing installs pick it up on update via the `load_config()` merge — no migration code. The local `voice`/`rate` keys stay as the `say` fallback; `edge_rate` (percent) tunes edge speed separately
- All hooks are silent no-ops when `~/.claude/simple-tts-config.json` doesn't exist — enabling the plugin without running setup does nothing
- `speak()` detaches the TTS process (`Popen` + `start_new_session`) so hooks return immediately and the 5 s hook timeout never cuts speech off. It builds the full payload (text, voice, earcon, …) **before** entering the state-file critical section, then does exactly one thing under `_locked_state`: kill+clear (priority), enqueue (busy, non-priority), drop (finished <2 s ago, dedup) or spawn. `_spawn` is the single `Popen` caller for both engines now — **`engine=say` also goes through `edge_speak.py`** (payload `{"engine": "say", ...}`) instead of `speak()` shelling out to `say` directly, so `say` shares the helper's queue-drain loop too; `speak_cli`/MCP `server.py` inherit this for free since they call `speak()`. `_spawn` leaves state untouched on a failed spawn (returns `None`, which `_locked_state` treats as "don't write")
- **Global speech queue**: when TTS is already busy, a non-priority `speak()` call is **queued** instead of dropped — a behavior change from before. `priority=True` (Notification hook) kills the running TTS process group AND clears the whole queue (`_queue_clear()`) before speaking, so a real interrupt doesn't get followed by a pile of stale queued messages. The narrow "just finished <2 s ago" dedup window still drops rather than queues. `_is_our_tts` checks the PID's command — a `say` process or the `edge_speak.py` helper — never other apps' `say`
- Stop hook only speaks when a `<!-- TTS: -->` (or categorized `<!-- TTS[ok|err|q]: -->`) tag exists — silence means Claude stopped for a permission prompt, and the notification hook handles that
- Phonetic replacement is whole-word, longest-match-first (so `deployed` doesn't get mangled by the `deploy` entry)
- `speak()` honors `"enabled": false` (mute) and `"quiet_hours"` (`{"start","end"}`, may wrap past midnight; malformed config fails open — speech stays on). `force=True` bypasses both (used only by speak_cli)
- **Voice howl vs distortion vs overlay are three independent switches** (decoupled from the old single `knight_rider`): (1) **howl** — the KITT siren bed under the voice — is `"voice_howl"` (`auto`|`on`|`off`, default `auto`); `auto` plays it **only when `overlay_theme == "kitt"`**, so every other theme (spark, hal, …) speaks without the siren. `tts_utils._howl_on(config)` resolves it and gates `intro_sound` in the edge payload (off ⇒ `"none"`). (2) **distortion** — the KITT −20 Hz pitch shift — is `"voice_distortion"` (`auto`|`on`|`off`, default `auto`; legacy bool accepted); `auto` follows the theme via `_distortion_on()` — on for the KITT-family (`kitt`, `cylon`), **off for `spark`** (plain voice) — gating `edge_pitch` (`-20Hz`/`+0Hz`); independent of the howl. Net effect of the auto/auto defaults per theme: **kitt** = howl + distortion, **cylon** = distortion only, **spark** = plain voice. (3) **overlay** — the floating scanner — stays on `"knight_rider"` (default `true`, `kitt_state.overlay_enabled`). Toggles: `/simple-tts:tts howl [on|off|auto]`, `/simple-tts:tts distortion [on|off|auto]`. The overlay's look **and** its on/off are both controlled by `/simple-tts:tts theme [name|off]` (a theme name sets `overlay_theme` + `knight_rider:true`; `off` sets `knight_rider:false`); default `overlay_theme` is `"spark"`, unknown names fall back to spark. There is no `knight-rider` subcommand anymore. The overlay is a separate daemon, so muting speech (`enabled:false`) does not hide it and vice-versa
- **Audio-reactive "speaking" eye** (`payload["envelope"]`): the overlay's speak animation is driven by the RMS envelope of the plain voice, written to `~/.claude/simple-tts-speak.json` by `edge_speak._play_speech`. The envelope is produced for **every** theme whenever the overlay is on (`speak()` sets `envelope = knight_rider`), decoupled from the siren: earlier it was gated by `intro_sound != none`, so spark/cylon (no siren) never got one and the host fell back to a synthetic `0.5 + 0.5·sin` pulse — the eye "pulsed" instead of reacting. Cost is paid once per phrase: the edge engine caches the envelope as a `<hash>.env.json` sidecar next to the audio (`audio_cache.read_env`/`store_env`), so repeats skip the ffmpeg pass; `say` (uncached fallback) computes it live; the overlay-off case skips the work entirely. The theme's response curve is unchanged (spark: `resp = level**0.6`, eye length ∝ `sy`) — only the signal feeding it went from synthetic to real
- **Attention mode**: the Notification hook marks "session waits for the user", the overlay blinks amber until the user reacts. Cleared by whichever comes first: a new prompt (UserPromptSubmit), a tool executing (PostToolUse — permission was granted mid-turn) or the turn ending (Stop). Markers go stale after 10 min so an orphaned marker (killed session) cannot blink forever
- **Busy-aware notification suppression**: `tts_utils.session_busy_fresh(session_id, max_age=BUSY_STALE_SECS)` (15 min, same window as the overlay's `think` mode) tells whether the session's turn is still in progress. During an in-progress turn the only genuinely user-actionable events are a `permission` notification (always spoken) and `AskUserQuestion`/`ExitPlanMode` (announced immediately by the `ask_question_tts.py` PreToolUse hook, which fires as the tool call happens, before any notification could go stale) — so it's safe for the Notification hook to suppress every other "waiting" notification while `session_busy_fresh()` is true; a hard-killed session's stale-but-fresh-looking busy marker can at worst mute a real "waiting" for up to 15 min, which is accepted
- **Project-name announce** (`"announce_project"` config key, `auto`|`on`|`off`, default `auto`): when multiple sessions run in parallel it can be unclear which one just spoke, so `speak()` can prepend a `"<project label>: "` prefix (project label = `_project_label()` of the hook's `cwd` basename, separators `-_.` turned into spaces) before sanitization — it replaces the name prefix for that message, it doesn't stack with it. `auto` (`_should_announce_project`) prefixes when **at least one OTHER session** is fresh-busy: `fresh_busy_count(exclude_session_id=session_id) > 0`, excluding the speaking session's own marker regardless of whether it happens to be set at the moment — `on` always prefixes (when the hook passed a `project`), `off` never does. `stop_tts`/`notification_tts`/`ask_question_tts` all pass `project=os.path.basename(cwd)` and `session_id=` (their hook's `session_id`) through to `speak()`. This fixes an earlier inconsistency: `fresh_busy_count()` used to count markers on disk with no exclusion and threshold `> 1`, so the Stop hook — which clears its own busy marker (`set_session_busy(..., False)`) before calling `speak()` — under-announced with exactly 2 parallel sessions (its own already-cleared marker meant `fresh_busy_count()` was counting *only* the other session, landing at 1, not > 1), while `notification_tts`/`ask_question_tts` (marker still set) counted themselves too and behaved differently. Excluding by `session_id` with threshold `> 0` makes all three hooks consistent regardless of whether the caller's own marker is up or down
- **Tag categories + earcons**: the tag grew an optional bracketed category, `<!-- TTS[ok|err|q]: message -->` — no brackets keeps today's neutral behavior, so it's fully backward compatible. `tts_utils.parse_tts_tag`/`TTS_TAG_RE` is the single place that parses it (lowercase only, no `IGNORECASE`; an unrecognized bracket like `[foo]` doesn't match the tag at all, treating it as absent — deliberate, see tests). Only `stop_tts.py` (tag mode) ever sets `category`; `notification_tts`/`ask_question_tts` speak with `priority=True` and carry no category/earcon. The `"earcons"` config key (default `true`) gates whether a recognized category actually reaches the payload as an `"earcon"` field — `speak()` computes `earcon = category if category in ("ok","err","q") and config.get("earcons", True) else None`. Playing the sound is `edge_speak._play_earcon`'s job (`hooks/sounds/earcon_ok.mp3`/`earcon_err.mp3`/`earcon_q.mp3`, played first and blocking, for both engines), independent of the audio cache and the KITT mix

## CI/CD

Every push to `main` triggers `.github/workflows/bump-version.yml`:
1. Determines bump type from commit message prefix: `feat:` → minor, `feat!:`/`breaking` → major, everything else → patch
2. Bumps version in `plugin.json`, commits with `[skip ci]`, creates git tag
3. Sends `repository_dispatch` to `usterk/simple` marketplace to update version and `ref`

**Commit message conventions**: prefix with `feat:` for minor bump, `feat!:` or `breaking` for major. No prefix = patch.
